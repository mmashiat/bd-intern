# Product Knowledge

> **Instructions**: Replace this file with detailed knowledge about your products.
> This file is the ground truth for all sales enablement content.
> The sales-enablement skill will ONLY make claims that are documented here.

## How to Fill This Out

For each product in your `config/company.yaml`, document:

### Product Name

**The Problem**
- What market problem does this product solve?
- How big is the problem? (TAM, affected users, cost of status quo)
- Who feels this pain most? (segments, personas)

**The Solution**
- What does your product do? (2-3 sentences, plain language)
- How does it work? (technical architecture, key mechanisms)
- What makes it different from alternatives?

**Key Capabilities**
- Capability 1: description + specific metric
- Capability 2: description + specific metric
- Capability 3: description + specific metric

**Technical Specs**
- Performance characteristics
- Integration requirements
- Supported platforms/environments
- Security model

**Use Cases**
For each use case:
- Target persona/segment
- Problem → Solution flow
- Expected outcome (quantified)

**Competitive Positioning**
- Direct competitors and how you differ
- Common objections and responses
- Where you win, where they win

**Proof Points**
- Customer deployments or case studies
- Performance benchmarks
- Third-party validations

**Sales Motion**
- Standard engagement offer (should match config)
- Typical timeline
- Key leverage points
- Contact info
